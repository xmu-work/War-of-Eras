# 战线进化：纪元之战 - 项目结构说明

这份结构是按 Unity 工程整理过的。根目录只保留仓库和工程配置，真正的游戏内容放在 `Assets/` 里。

## 目录含义

### `Assets/`
Unity 识别的主资源目录。

- `Art/`：角色、塔、防御设施、背景、界面美术
- `Audio/`：音效和背景音乐
- `Scenes/`：场景文件，当前主入口是 `Battle.unity`
- `Scripts/`：脚本代码
- `Prefabs/`：预制体
- `Data/`：数值配置和规则数据
- 其他目录：Unity 导入的模板资源或编辑器相关资源

### `Packages/`
Unity 包管理配置。

- `manifest.json`：项目依赖
- `packages-lock.json`：依赖锁定

### `ProjectSettings/`
Unity 工程设置。

- 图形、输入、物理、质量、构建等配置都在这里

### `素材包/`
原始素材和拆分资源，属于素材来源区。

### `tmp/`
临时处理目录，后续不需要时可以清理。

## 现在这样做的好处

1. 你可以直接用 Unity Hub 打开仓库根目录。
2. Git 仓库不需要换，Unity 生成的内容会跟着当前仓库走。
3. 后续新增内容只需要往 `Assets/` 里放，不用再维护根目录那套旧结构。

## 建议的实际使用方式

- `Battle.unity` 作为主战斗场景，内部用 `MetaGameController` 切主菜单和游戏画布
- 后续如果确实要独立主菜单或结算界面，再单独加新场景
- 脚本放 `Assets/Scripts/`
- 图像和 UI 放 `Assets/Art/`
- 音效放 `Assets/Audio/`
- 数值表和规则放 `Assets/Data/`

这样后面做关卡、单位、炮塔、技能和时代扩展时，结构会比较干净。
