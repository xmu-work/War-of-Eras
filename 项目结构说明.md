# 战线进化：纪元之战 - 项目结构说明

这个结构的思路是：**代码按系统分，素材按类型分，数值按数据分**。这样后面加新时代、新兵种、新炮塔时，不容易乱。

## 当前目录

```text
Assets/
  Art/
    Units/
    Towers/
    Bases/
    Backgrounds/
    VFX/
    UI/
  Audio/
    SFX/
    Music/
Data/
  Ages/
  Units/
  Towers/
  Skills/
  Balance/
Scripts/
  Core/
  Combat/
  Economy/
  Era/
  Towers/
  Skills/
  UI/
Scenes/
```

## 每一层代表什么

### `Assets`
放所有美术和音频资源。

- `Art/Units/`：兵种图片、立绘、行走帧、攻击帧
- `Art/Towers/`：炮塔外观、炮口、炮弹相关图片
- `Art/Bases/`：基地外观、升级后的基地图
- `Art/Backgrounds/`：战场背景、时代背景
- `Art/VFX/`：技能特效、爆炸、闪电、灼烧等效果
- `Art/UI/`：按钮图标、面板底图、资源图标
- `Audio/SFX/`：攻击、受击、爆炸、升级等音效
- `Audio/Music/`：背景音乐和时代音乐

### `Data`
放数值配置和规则数据。

- `Ages/`：每个时代的解锁条件、升级收益、时代技能
- `Units/`：兵种属性表，比如生命、攻击、射程、速度、训练时间
- `Towers/`：炮塔属性表，比如费用、伤害、射程、特殊效果
- `Skills/`：基地技能、时代大招、冷却时间、效果参数
- `Balance/`：整体数值规则、平衡表、成长曲线、测试记录

### `Scripts`
放程序代码，按职责拆开。

- `Core/`：游戏主流程，比如开局、暂停、胜负、状态切换
- `Combat/`：战斗逻辑，比如单位移动、索敌、攻击、伤害计算
- `Economy/`：金币、时代值、收入、返还、消耗
- `Era/`：时代升级、进化选择、时代切换
- `Towers/`：炮塔建造、移动、合成、射击逻辑
- `Skills/`：技能释放、冷却、持续效果
- `UI/`：界面按钮、面板、提示、数值显示

### `Scenes`
放场景文件。

- `MainMenu`：主菜单
- `Battle`：对战主场景
- `Result`：胜负结算

## 这个结构为什么好用

1. 新加时代时，只需要往 `Assets/Art`、`Data`、`Scripts` 里对应时代补内容。
2. 平衡数值时，只改 `Data`，不用到处翻代码。
3. 程序员和美术可以分开做，不容易互相打架。
4. 后期如果要做第五时代、第六时代，也只是继续加目录，不用推翻重来。

## 你接下来可以怎么用

- 先把第一时代的兵种、炮塔、基地素材放进对应目录。
- 再把数值表写进 `Data/`。
- 最后在 `Scripts/` 里把核心战斗流程接起来。
