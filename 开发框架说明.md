# 战线进化：纪元之战 - 开发框架说明

本文档给团队成员说明当前项目应该怎么开发。目标是先做出一个主体框架，再让不同成员分别完善不同时代的内容。

## 一句话框架

核心代码只负责“规则怎么运行”，时代成员只负责“这个时代有哪些兵、塔、技能和表现”。

也就是说，主体框架不要写死蛮荒、机械、电力、核能、星海的具体数值。具体内容尽量放在 `Assets/Data/` 和 `Assets/Prefabs/` 里。

## 主体玩法闭环

第一阶段要优先跑通这个闭环：

```text
进入 Battle.unity
  -> 初始化双方基地、三条线路、资源井和炮位
  -> 玩家选择线路并消耗金币出兵
  -> 单位沿线路移动、索敌、攻击、死亡
  -> 金币随时间增长，击杀返还金币
  -> 时代值增长，满后玩家选择进攻或防守倾向升级
  -> 新时代解锁新兵种、新炮塔、新技能和基地属性
  -> 摧毁敌方基地获胜，己方基地被摧毁失败
```

主体没跑通前，不建议直接堆五个时代的完整内容。

## 脚本分层

新的玩法脚本统一放在 `Assets/Scripts/Battle/`。

```text
Core/
  对局生命周期、胜负状态、全局事件、暂停和重开。

Combat/
  生命、伤害、攻击、索敌、状态效果、死亡处理。

Economy/
  金币、基础收入、时代收入加成、资源井收入、击杀返还。

Progression/
  时代值、升级门槛、时代切换、进攻/防守倾向 Buff。

Towers/
  炮位、建塔、炮塔索敌、攻击、拖动换位、合成升星。

Map/
  上中下三路、线路节点、连接点、据点、资源井归属。

AI/
  敌方出兵节奏、路线选择、建塔和技能释放策略。

UI/
  金币、时代值、基地血条、线路选择、出兵按钮、技能冷却、升级弹窗。

Data/
  ScriptableObject 数据定义、数据加载、公共枚举。
```

旧的 Platformer 模板脚本、教程、角色、关卡素材和预制体已经清理。新的玩法代码统一从 `Assets/Scripts/Battle/` 扩展。

## 数据目录

数值和规则配置放在 `Assets/Data/`。

```text
Assets/Data/Ages/       # 时代配置
Assets/Data/Units/      # 兵种配置
Assets/Data/Towers/     # 炮塔配置
Assets/Data/Skills/     # 技能配置
Assets/Data/Balance/    # 全局平衡参数
```

每个内容目录下按时代分文件夹：

```text
Tribe/      # 蛮荒部落
Machine/    # 机械工坊
Electric/   # 电力时代
Nuclear/    # 核能纪元
Starsea/    # 星海文明
```

建议后续用 ScriptableObject 建这些数据类型：

```text
AgeDefinition
  时代编号、时代名称、基地生命、金币收入加成、可用兵种、可用炮塔、时代大招、进攻倾向、防守倾向。

UnitDefinition
  名称、时代、费用、生命、伤害、攻速、攻击范围、移动速度、训练时间、特殊效果、预制体。

TowerDefinition
  名称、时代、费用、伤害、攻速、射程、星级倍率、特殊效果、预制体。

SkillDefinition
  名称、类型、冷却、伤害、范围、持续时间、状态效果、特效预制体。

BalanceDefinition
  初始金币、基础收入、时代门槛、击杀返还比例、炮塔扩展费用、通用倍率。
```

## 预制体目录

最终由脚本生成或场景引用的预制体放在 `Assets/Prefabs/`。

```text
Assets/Prefabs/Units/
Assets/Prefabs/Towers/
Assets/Prefabs/Bases/
Assets/Prefabs/VFX/
Assets/Prefabs/UI/
```

单位成员提交一个兵种时，至少要有：

- 一个 `UnitDefinition`
- 一个单位 Prefab
- 单位显示用 Sprite 或动画
- 至少一轮游戏内测试

炮塔成员提交一个炮塔时，至少要有：

- 一个 `TowerDefinition`
- 一个炮塔 Prefab
- 攻击表现或占位特效
- 至少一轮游戏内测试

## 时代成员交付清单

每个时代建议最终交这些内容：

```text
3 个兵种
3 个炮塔
1 个时代大招
1 个进攻倾向
1 个防守倾向
基地、资源井、据点或 UI 的时代视觉变化
简单测试说明：强项、弱项、是否有明显超标
```

前期可以先降级为：

```text
2 个兵种
1 个炮塔
1 个时代大招
一套临时美术或占位 Prefab
```

## 开发顺序

推荐顺序如下：

1. 核心组先做三路战斗、单位移动、索敌、攻击、基地胜负。
2. 加金币收入、出兵按钮和线路选择。
3. 加时代值和“蛮荒 -> 机械”的升级流程。
4. 加固定炮位、建塔和炮塔攻击。
5. 加资源井、前线据点和基地技能。
6. 框架稳定后，其他成员并行补电力、核能、星海。
7. 最后统一做 UI polish、特效、音效、平衡和打包。

## 合并规则

- 改核心脚本前，先和核心组说一声。
- 每个人尽量只改自己负责时代的目录。
- 新增文件命名尽量使用中文显示名 + 清晰英文目录，例如 `Nuclear/核能坦克.asset`。
- 不要在场景里堆大量临时对象；能做成 Prefab 的先做成 Prefab。
- 不要把测试素材直接丢到 `Assets/` 根目录。

## 当前状态

当前工程保留 Unity 项目外壳、渲染设置、TextMesh Pro、`Battle.unity` 主场景和 `Assets/Scripts/Battle/` 新框架入口。旧 Platformer 模板层已清理，后续应直接围绕三路线自动战斗、经济、时代、炮塔和 UI 继续开发。
