# 战线进化：纪元之战 - 项目结构说明

这份结构按“多人协作 + 数据驱动”整理。根目录只放仓库说明、Unity 工程配置和原始素材，真正进入 Unity 的游戏内容都放在 `Assets/` 里。

## 当前目录

```text
War-of-Eras/
  Assets/
    Art/                 # 最终进入游戏的美术资源
    Audio/               # 音效和背景音乐
    Data/                # 时代、兵种、炮塔、技能、平衡数据
    Prefabs/             # 单位、炮塔、基地、特效、UI 预制体
    Scenes/              # 场景文件，主入口是 Battle.unity
    Scripts/
      Battle/            # 新的战斗策略游戏主体框架
    Rendering/
    Settings/
    TextMesh Pro/
  Packages/
  ProjectSettings/
  素材包/                # 原始素材和拆分资源，不直接作为最终结构
  tmp/                   # 临时处理目录
  README.md
  开发框架说明.md
  项目结构说明.md
  .gitignore
```

## 核心原则

项目现在分成两类内容：

- 核心框架：由少数人维护，负责战斗循环、单位行为、经济、时代升级、炮塔、地图节点、AI、UI 控制。
- 时代内容：由多人并行制作，分别负责蛮荒、机械、电力、核能、星海的数值、预制体、美术和特效。

后续做功能时，优先把可变化的内容放进 `Assets/Data/`，不要把某个时代、某个兵种或某座塔的数值写死在脚本里。

## 重点目录说明

### `Assets/Scripts/Battle/`

新的主体玩法代码放这里。旧 Platformer 模板脚本已经清理，不再作为项目结构的一部分。

```text
Assets/Scripts/Battle/
  Core/          # 对局生命周期、全局状态、事件调度
  Combat/        # 单位生命、攻击、伤害、状态效果
  Economy/       # 金币收入、击杀返还、资源井收益
  Progression/   # 时代值、时代升级、进攻/防守倾向
  Towers/        # 炮位、建塔、炮塔攻击、移动、合成
  Map/           # 三路线路、节点、据点、资源井
  AI/            # 敌方出兵、路线选择、技能释放策略
  UI/            # 战斗 HUD、按钮、弹窗、冷却显示
  Data/          # ScriptableObject 定义和运行时数据读取
```

### `Assets/Data/`

数值与规则配置放这里。团队成员负责某个时代时，优先改这里。

```text
Assets/Data/
  Ages/          # 每个时代的基础配置
  Units/         # 兵种数据
  Towers/        # 炮塔数据
  Skills/        # 时代大招和基地技能
  Balance/       # 全局经济、时代门槛、倍率表
```

每个时代使用固定英文目录名：

- `Tribe`：蛮荒部落
- `Machine`：机械工坊
- `Electric`：电力时代
- `Nuclear`：核能纪元
- `Starsea`：星海文明

### `Assets/Prefabs/`

最终可拖进场景或由脚本生成的预制体放这里。

```text
Assets/Prefabs/
  Units/         # 单位预制体
  Towers/        # 炮塔预制体
  Bases/         # 基地、据点、资源井
  VFX/           # 技能、攻击、受击、死亡特效
  UI/            # UI 组件预制体
```

### `Assets/Art/`

最终美术资源放这里。`Assets/Art/Generated/` 可以继续保留生成素材和候选图，真正投入使用的拆分图、图标、序列帧建议整理进 `Units`、`Towers`、`Bases`、`VFX`、`UI` 等目录。

### `素材包/`

这是原始素材来源区，可以放大图、候选图、生成过程文件或未整理素材。最终被 Unity 使用的资源，应复制或整理到 `Assets/Art/`、`Assets/Audio/`、`Assets/Prefabs/` 中。

## 多人协作规则

1. 核心框架成员主要改 `Assets/Scripts/Battle/`。
2. 时代内容成员主要改自己时代的 `Assets/Data/`、`Assets/Prefabs/`、`Assets/Art/`。
3. 不要在 `Assets/` 根目录散放测试素材；先归到 `Art`、`Data`、`Prefabs` 或 `Scripts/Battle`。
4. 新增兵种、炮塔和技能时，先建数据，再建预制体，最后接入 UI 和测试。

更详细的开发接口和交付清单见 [开发框架说明.md](开发框架说明.md)。
