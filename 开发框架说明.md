# 战线进化：纪元之战 - 开发框架说明

本文档说明当前玩法框架如何工作，以及后续扩展时应该优先改哪些位置。

## 核心闭环

当前游戏闭环如下：

```text
主菜单选择地图和难度
  -> 进入 Battle 场景
  -> 初始化地图、基地、路线、设施槽位和 HUD
  -> 玩家出兵或派遣建筑兵
  -> 单位沿路线推进、索敌、攻击单位/设施/基地
  -> 金币和时代值持续增长
  -> 玩家修建防御塔、资源点，释放时代技能
  -> 时代值满后选择进攻或防守进化
  -> 摧毁敌方基地胜利，己方基地被摧毁失败
```

## 代码职责

### `BattleGameController`

这是战斗场景的总调度器，主要负责：

- 战场、基地、设施槽位、HUD 和音频的运行时生成
- 金币、收入、时代值、冷却和胜负状态
- 玩家出兵、框选单位、路线重定向
- 建筑兵派工、待建任务、设施落地
- 敌方 AI 出兵、建塔、建资源点和时代推进
- 时代升级后的数值、地图、基地和防御塔刷新

### `BattleRuntimeActors`

这里放运行时对象：

- `BattleUnit`：士兵和建筑兵的移动、攻击、状态、选中环、建筑兵工作效果
- `BattleTower`：防御塔生命值、攻击、受损变色、修复
- `BattleResourceWell`：资源点生命值、受损变色、修复
- `BattleFacility`：防御塔和资源点共享的抽象接口

### `GameSetup`

负责主菜单和战斗之间共享的轻量配置：

- 当前地图
- 当前难度
- 难度对应的初始金币、收入、敌方生命/伤害/出兵节奏

### `MainMenuController`

主菜单 UI 也是运行时生成的，负责：

- 地图选择
- 难度选择
- 开始游戏
- 教程、图鉴、设置等占位信息页

## 建筑兵机制

建筑兵通过 `UnitRole.Builder` 与普通战斗兵区分。

修建设施时：

1. 玩家点击防御塔或资源点按钮。
2. 控制器显示可建造槽位。
3. 玩家点击槽位后，控制器扣金币并记录待建任务。
4. 控制器为建筑兵计算到目标点的最短路线。
5. 建筑兵到达施工范围后调用 `TryCompleteBuilderTask()`。
6. 控制器生成对应的防御塔或资源点。

修复设施时：

1. 建筑兵没有待建任务。
2. 建筑兵在自身工作范围内寻找受损友方设施。
3. 找到后逐帧调用设施的 `Repair()`。
4. 设施恢复生命值并更新受损颜色。

## 路线系统

项目有两套路线路径：

- `LaneRoutes`：三条主路，用于默认出兵。
- `RouteNodes` + `RouteEdges`：整张地图的路线图，用于框选单位重定向和建筑兵寻路。

建筑兵会从三条出生线中选择总距离最短的路线，再补最后一小段到目标设施中心。框选单位则会从当前位置吸附到路线图，再向玩家点击的目标点寻路。

## 扩展兵种

新增兵种时建议按这个顺序：

1. 把单位序列帧放入对应时代的 `Resources/时代目录/Units/单位名/`。
2. 在 `AgeVisualSets` 里确认单位目录名。
3. 在 `BuildDefinitions()` 中补 `UnitDefinition`。
4. 如果是建筑兵，把角色设为 `UnitRole.Builder`。
5. 进入战斗场景测试出兵、移动、攻击或施工。

## 扩展防御塔

新增防御塔时建议按这个顺序：

1. 把攻击序列帧放入 `Resources/时代目录/Towers/塔名/`。
2. 在 `AgeVisualSets` 的 `towerFramePrefixes` 中补路径前缀。
3. 在 `BuildTowerDefinitions()` 中补 `TowerDefinition`。
4. 测试建造、攻击、被攻击、修复和时代升级后的视觉刷新。

## 扩展地图或设施槽位

当前控制器内置了默认坐标。若要让场景覆盖这些坐标，可在 `Battle` 场景中使用 `BattleLayout` 约定的层级：

```text
Layout/
  Bases/
    PlayerBasePoint
    EnemyBasePoint
  Routes/
    Lane_0/
    Lane_1/
    Lane_2/
  Towers/
    PlayerTowerSlot_*
    EnemyTowerSlot_*
  ResourceWells/
    PlayerWellSlot_*
    EnemyWellSlot_*
```

启动时 `BattleLayout.Validate()` 会检查关键标记。如果缺少标记，控制器会回退到代码里的默认坐标。

## 验证方式

当前没有自动化测试用例，建议每次功能改动后：

1. 用 Unity 打开项目，确认 C# 编译无错误。
2. 从 `MainMenu` 进入 `Battle`，测试出兵、框选、建塔、建资源点、建筑兵修复、时代升级和胜负结算。
3. 命令行运行 `git diff --check`，检查明显格式问题。
